# 🚀 Learning-API

Welcome to my API learning repository! This project tracks my journey learning the core basics of working with LLM APIs and building simple agents with function calling. 

All scripts and notebooks here are fully optimized for Groq's API using their `chat.completions` architecture.

---

## 🛠️ What I Learned

This repository follows my step-by-step learning progression:

| # | Module | Description |
|---| ------ | ----------- |
| 1 | `text_prompt` | Structuring basic system and user prompts. |
| 2 | `joke_history` | Managing chat history and conversational memory. |
| 3 | `counting_tokens` | Understanding token limits and counting overhead. |
| 4 | `managing_tokens` | Strategies for optimizing and staying within token budgets. |
| 5 | `streaming` | Implementing real-time streaming responses. |
| 6 | `rate_limits` | Handling API rate limits and backoff strategies gracefully. |
| 7 | `structured_outputs` | Enforcing JSON schemas and structured LLM responses. |
| 8 | `tool_calling` | Giving the model capabilities to call external functions (Agents). |
| **9** | **intelligent_inbox_agent** | **[PROJECT] Built a fully autonomous multi-turn email triage pipeline. Features multi-stage AI orchestration, Pydantic type-safety validation, dynamic hyperparameter execution (altering model temperature based on classified urgency), and structured multi-turn tool calling error-handling loops.** |
